Policy

Djecrety is hosted on Github Pages and doesn't have any back-end code or database.
The keys generate on demand and it's completely random and never store to any file or database.

That's all. So you can safely use Djecrety.

Got It!
Why we must change the secret key?

As Django official documentation says:

A secret key used to provide cryptographic signing and should be set to a unique, unpredictable value.
...
Running Django with a known SECRET_KEY defeat many of Djangoâ€™s security protections, and can lead to privilege escalation and remote code execution vulnerabilities.

So recommended: Keep it safe. Ignore 'settings.py' file in your commits. Change the secret key on your deploy. If you lost the secret key for any reason or the server got compromised change it as soon as possible.

Got It!
How to use?

After creating your project:

1. Duplicate your project 'settings.py' file and rename it to 'settings.py.sample'.

2. Open 'settings.py.sample' file and set empty SECRET_KEY value (line 23).

SECRET_KEY = ''

3. Add 'settings.py' file to '.gitignore' file. (If using git)

4. When you want to deploy your project on the server (production or test), duplicate 'settings.py.sample' file.

5. Rename duplicated file to 'settings.py'.

6. Set SECRET_KEY value to the new secret key generated with Djecrety in 'settings.py' file.
