Policy

Djecrety is written using pure Javascript, CSS and HTML and do not have any back-end code and database. The keys generate on demand and is completely random and never store to any file or database.

So you can safely use Djecrety.

Got It!
Why we must change secret key?

As Django official documentation says:

A secret key used to provide cryptographic signing, and should be set to a unique, unpredictable value.

Running Django with a known SECRET_KEY defeats many of Djangoâ€™s security protections, and can lead to privilege escalation and remote code execution vulnerabilities.

So it's recommended: Ignore settings.py file in your commits, change the secret key on your deploy. If you lost the secret key for any reason or the server got compromised change it as soon as possible.

Got It!
How to use?

After create your project:


1. Duplicate your project 'settings.py' file and rename it to 'settings.py.sample'.

2. Open 'settings.py.sample' file and set empty SECRET_KEY value (line 23). 

SECERET_KEY = ''

3. Add 'settings.py' file to '.gitignore' file. (If git used)

4. When you want to deploy your project on server (production or test), duplicate 'settings.py.sample' file.

5. Rename duplicated file to 'settings.py'.

6. Generate a new secret key and set SECRET_KEY value to the new secret key in 'settings.py' file.
